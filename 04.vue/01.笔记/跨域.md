# 跨域
违背了同源策略（协议名、域名、端口号三者必须完全一致）

## 解决跨域
1. jsonp
利用script天然跨域的特性解决跨域
问题：只能发送GET请求

2. cors
官方的方案，通过http协议来解决
设置响应头 access-control-allow-origin 
能支持任意方式请求

3. 服务器代理
- 正向代理
  - 配置webpack的devServer
  - proxy
  ```
  proxy: {
    "/api": {
      // 以 /api 开头的请求
      target: "http://localhost:3000", // 目标服务器地址
      pathRewrite: {
        // 重写路径
        "^/api": "", // 去掉/api
      },
      // changeOrigin: true
    }
  }
  ```
- 反向代理 nginx
